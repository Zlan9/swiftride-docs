# 📘 SCM BYOD-3  
## Collaborative Git Workflow at CodeNest Technologies

---

## 📌 Overview  
This project simulates a real-world collaborative Git workflow using two separate GitHub accounts operating in different environments. It demonstrates how developers collaborate on a shared repository using branching strategies, pull requests, code reviews, and conflict resolution.

The scenario is based on CodeNest Technologies building documentation for a ride-sharing application called **SwiftRide**.

- **Account A** → Repository Owner (Primary Machine)  
- **Account B** → Collaborator (WSL/VM Environment)  

---

## 🎯 Objectives  
- Understand multi-user collaboration using Git and GitHub  
- Practice branching and pull request workflows  
- Perform structured code reviews  
- Resolve merge conflicts  
- Maintain clean commit history  

---

## 🛠️ Pre-requisites  
- Two GitHub accounts  
- Git installed in both environments  
- Separate Git configurations for each account
